Detailed Description

Post-order traversal analysis for functions.

Before use this analysis you have to run doFuncsAnalysis().

For this example below analysis returns functions in this order: funcFirst, funcSecond and funcTop. As you can see in post-order traversal.

define void @funcTop() {
call void @funcFirst()
call void @funcSecond()
ret void
}
define void @funcFirst() {
ret void
}
define void @funcSecond() {
ret void
}

Functions can be in strongly connected component. Example below shows this situation. In this situation is not possible to do a correct post-order. It is because functions sccFunc and sccFunc1 creates strongly connected component. You can use here two types of traversal:

  1. When you use for getting next function only method getNextFunc(), than you get functions in this order: endFunc, sccFunc1, sccFunc, funcTop. All functions only once.

  2. When you are sure by isNextInSCC() that the next function is in strongly connected component than you can use getNextFuncInSCC(). This method causes iterating through strongly connected component until you use method stopIteratingSCC(). Then use getNextFunc() which returns the next function which is out from iterated strongly connected component.

    For example order:

    1. getNextFunc() - returns endFunc.
    2. isNextInSCC() => returns true then getNextFuncInSCC() - returns sccFunc1.
    3. 2 times getNextFuncInSCC() - returns sccFunc and sccFunc1.
    4. stopIteratingSCC() and getNextFunc() - returns funcTop.
      define void @funcTop() {
      call void @sccFunc()
      ret void
      }
      define void @sccFunc() {
      call void @sccFunc1()
      ret void
      }
      define void @sccFunc1() {
      call void @sccFunc()
      call void @endFunc()
      ret void
      }
      define void @endFunc() {
      ret void
      }

Function * bin2llvmir::FuncTraversalAnalysis::getNextFuncInSCC ( )

Returns next function that is part of some SCC.

You have to use this method only if you know that next returned function will be part of some SCC. For this purpose to check use method isNextInSCC(). Than you can use this method more than once. To stop iterating through functions in some SCC use stopIteratingSCC().
