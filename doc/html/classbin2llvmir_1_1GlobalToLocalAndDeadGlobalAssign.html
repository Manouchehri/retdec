Detailed Description

Global to local and dead global assign optimization.

What optimizes each optimization

See also
GlobalToLocal and
DeadGlobalAssign.

We don't optimize global variable in some special cases:

  1. Global variable is not single type (array, structure, etc.).
  2. Global variable is a pointer.
  3. Address of global variable can be taken.
  4. Global variable that doesn't have private or internal linkage.

To decide when something optimize this optimization uses three pieces of information that are obtained in this optimization:

  • Extended Right uses (in code extRUses)
  • Last left uses (in code lastLUses).
  • Not go through global variables (in code notGoThrough). How is this info obtained and what means
    See also
    StoreLoadAnalysis.

void bin2llvmir::GlobalToLocalAndDeadGlobalAssign::convertGlobsToLocUseInOneFunc ( const GlobVarSet &  globs)

Converts global variables in globs to local in specific situation which is mentioned below.

int g = 2;
func() {
x = g;
}

In this situation we can convert global variable to local but only if global variable g is not used in another function. We create new local variable for which we assign value 2.
