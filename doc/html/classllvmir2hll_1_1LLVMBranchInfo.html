Detailed Description

Supportive information about LLVM branches and loops.

This class contains supportive information about LLVM branches and loops. For example, it can compute the common branch destination of two basic blocks, which comes handy when generating if statements. It also provides several functions regarding loops.

Before an instance of this class can be used, the init() member function has to be called. More specifically, this member function has to be called whenever a new function is being decompiled.

Instances of this class have reference object semantics. This class is not meant to be subclassed.

llvm::BasicBlock * llvmir2hll::LLVMBranchInfo::findCommonBranchDestination ( llvm::BasicBlock *  bb1,
llvm::BasicBlock *  bb2 
) const

Returns the common branch destination of bb1 and bb2.

Given two basic blocks, bb1 and bb2, this function returns the common branch destination of branches in bb1 and bb2. If there is no common branch destination, the null pointer is returned.

For example, lets have the following piece of code.

if (cond) {
bb1:
...
goto lab;
} else {
bb2:
...
goto lab;
}
lab:
...

Then, for bb1 and bb2, this function returns lab.

Preconditions
  • both bb1 and bb2 are non-null

◆ findCommonSwitchDestination()

llvm::BasicBlock * llvmir2hll::LLVMBranchInfo::findCommonSwitchDestination ( llvm::SwitchInst *  si) const

Tries to find a common destination of branches in the given switch instruction.

For example, the following switch statement has bb as the common destination of branches:

switch (x) {
case 1:
...
break
case 2:
...
return
case 3:
...
break
}
bb

If there is no common destination, the null pointer is returned.

Preconditions
  • si is non-null

◆ getTripCount()

ShPtr< ConstInt > llvmir2hll::LLVMBranchInfo::getTripCount ( const llvm::Loop *  l) const

Returns the trip count of the given loop l.

Returns a loop-invariant constant integer indicating the number of times the loop will be executed. Note that this means that the backedge of the loop executes N-1 times. If the trip count cannot be determined, the function returns 0.

The IndVarSimplify pass transforms loops to have a form that this function easily understands.

◆ processAndPushBasicBlock()

void llvmir2hll::LLVMBranchInfo::processAndPushBasicBlock ( llvm::BasicBlock *  poppedBB,
BFSQueue &  bfsQueue,
BBSet &  bfsProcessedBBs 
) const
private

Processes the given basic block in a BFS search.

Checks whether poppedBB hasn't been traversed yet. If it hasn't, the function pushes its successors to bfsQueue and marks it as a processed basic block in bfsProcessedBBs. The only pushed successors are the ones that are not the header of a loop in which poppedBB is. This ensures proper code emission.

Preconditions
  • poppedBB is non-null
