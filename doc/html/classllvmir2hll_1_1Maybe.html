Detailed Description

template<typename T>
class llvmir2hll::Maybe< T >

A simple implementation of the Maybe monad from Haskell.

Informally speaking, this class may be used as the return type of functions which may either return the result or an "I don't know." answer.

The purpose of this class is similar to the purpose of the boost::optional<> template (see http://www.boost.org/doc/libs/1_53_0/libs/optional/doc/html/index.html). The referenced article also shows the motivation for such a class.

Usage example:

Maybe<int> getWinningLotteryNumber() {
if (...) {
// I know the answer.
return Just(42);
} else {
// I don't know the answer.
return Nothing<int>();
}
}
int main() {
Maybe<int> winningNumber = getWinningLotteryNumber();
if (winningNumber) {
// We have an answer.
std::cout << "The winning number is " << winningNumber.get() << "\n";
} else {
// No answer.
std::cout << "We don't know the winning number\n";
}
}

const T* llvmir2hll::Maybe< T >::operator-> ( ) const
inlinenoexcept

Returns a pointer to the value stored in this object.

When you have a Maybe<> object holding a value, you can use this operator to simplify the notation. For example, assume that Point is a structure containing attributes x and y, and consider the following variable:

Maybe<Point> p = Just(Point(1, 2));

Then, to access x and y, instead of

p.get().x
p.get().y

you can write

p->x
p->y
Preconditions
  • there is a value in this object, i.e. this object evaluates to true
